# ZombieAutoClicker - 项目智能文档

## 项目概述

**ZombieAutoClicker** 是一个针对游戏《向僵尸开炮》的Windows桌面自动化辅助工具。该项目使用C# .NET 8.0开发，结合计算机视觉（OCR）和图像识别技术，实现游戏界面的自动识别与操作。

## 核心功能

### 1. 游戏自动化挂机
- **元素试炼模式**：自动完成游戏中的元素试炼关卡
- **多模式支持**：支持雾境迷宫、寰球试炼、主线推图等模式（开发中）
- **智能状态机**：采用严格的两阶段状态机控制流程

### 2. 视觉识别系统
- **PaddleOCR集成**：基于深度学习的文字识别，高准确率
- **图像模板匹配**：支持PNG/JPG图片模板识别
- **混合识别引擎**：文字与图片混合识别，根据后缀自动选择识别方式

### 3. 用户界面
- **主控制面板**：模式选择、开始/停止控制、实时日志显示
- **悬浮窗显示**：实时显示OCR识别结果和边界框
- **自适应界面**：支持窗口大小调整，控件自动布局

## 技术架构

- **语言**：C# (.NET 8.0)

## 核心组件详解

### 1. GameBotController - 游戏机器人控制器
**位置**: `Controllers/GameBotController.cs`

**功能**：
- 实现严格的两阶段状态机控制
- 阶段1：进图准备阶段（顺序执行图片模板点击）
- 阶段2：战斗循环阶段（优先级循环执行文字识别点击）
- 智能重置机制：检测到返回主界面时自动重置状态

**关键特性**：
- 图文混合识别：根据目标后缀自动选择识别方式
- 相对路径支持：图片资源从exe同级目录加载
- 错误处理：完善的异常捕获和日志记录

### 2. VisionService - 视觉识别服务
**位置**: `Services/VisionService.cs`

**功能**：
- 封装PaddleOCR引擎，提供文字识别服务
- 实时OCR结果事件广播
- 智能目标中心点计算

**技术实现**：
- 静态OCR引擎实例，避免重复加载模型
- 支持中英文混合识别
- 返回识别文本的边界框和中心坐标

### 3. NativeMethods - 系统交互层
**位置**: `Interop/NativeMethods.cs`

**功能**：
- 窗口查找和捕获
- 屏幕截图功能
- 鼠标点击模拟

**Windows API封装**：
- `FindWindow`：查找游戏窗口
- `GetWindowRect`：获取窗口位置和大小
- `mouse_event`：模拟鼠标点击
- `SetCursorPos`：设置鼠标位置

### 4. MainForm - 主控制界面
**位置**: `MainForm.cs`

**功能**：
- 提供用户交互界面
- 模式选择（元素试炼、雾境迷宫等）
- 开始/停止控制
- 实时日志显示

**UI特性**：
- 纯代码生成控件，无设计器依赖
- 自适应布局，支持窗口缩放
- 黑色背景绿色文字的终端风格日志框

### 5. OverlayForm - 悬浮窗
**位置**: `OverlayForm.cs`

**功能**：
- 全屏透明悬浮窗，显示OCR识别结果
- 实时绘制文字边界框
- 显示识别到的文本内容

**视觉特性**：
- 透明背景，不影响游戏操作
- 红色边界框和文字标注
- 实时更新，反映当前识别状态

## 工作流程

### 自动化执行流程
1. **初始化**：启动程序，显示主界面和悬浮窗
2. **模式选择**：用户选择挂机模式（如"元素试炼"）
3. **开始挂机**：
   - 查找游戏窗口并获取屏幕截图
   - 阶段1：按顺序点击进图按钮（图片模板识别）
   - 阶段2：循环检测战斗界面文字并点击（OCR文字识别）
   - 智能重置：检测到返回主界面时自动重新开始
4. **停止挂机**：用户点击停止按钮，结束自动化流程

### 识别流程
1. **目标判断**：根据目标字符串后缀判断识别类型
   - `.png`/`.jpg`：图像模板匹配
   - 其他：OCR文字识别
2. **图像处理**：截取游戏窗口画面
3. **识别执行**：
   - 图片：使用OpenCV进行模板匹配
   - 文字：使用PaddleOCR进行文字识别
4. **坐标计算**：计算识别目标的中心点坐标
5. **点击执行**：转换为屏幕绝对坐标并模拟点击

## 配置与部署

### 环境要求
- Windows 10/11 操作系统
- .NET 8.0 Runtime
- 游戏《向僵尸开炮》窗口需要可见

### 资源文件
- **图片模板**：需要放置在`assets/`目录下
  - `assets/btn_start_element.png`：开始按钮
  - `assets/GameStart.png`：游戏开始按钮
- **OCR模型**：自动从`inference/`目录加载

### 编译与运行
1. **编译**：使用Visual Studio或`dotnet build`命令
2. **运行**：启动`ZombieAutoClicker.exe`
3. **配置**：确保游戏窗口标题为"向僵尸开炮"

## 开发约束与注意事项

### 技术约束
1. **窗口依赖**：必须找到标题为"向僵尸开炮"的游戏窗口
2. **图片路径**：图片模板必须放在exe同级目录的assets文件夹
3. **分辨率适配**：目前针对特定游戏分辨率优化，可能需要调整识别区域
4. **性能考虑**：OCR识别有一定性能开销，循环间隔需要合理设置

### 安全与合规
1. **仅限个人使用**：自动化工具可能违反游戏服务条款
2. **本地处理**：所有识别在本地完成，无网络传输
3. **无恶意行为**：仅模拟鼠标点击，不修改游戏内存

### 扩展性
1. **模式扩展**：可通过修改`GameBotController.Start()`方法添加新模式
2. **识别目标扩展**：在`_step1Texts`和`_step2Texts`数组中添加新目标
3. **UI扩展**：可通过修改MainForm添加新控件和功能

## 故障排查

### 常见问题
1. **找不到游戏窗口**：确保游戏已启动且窗口标题正确
2. **识别失败**：检查图片模板路径和文件是否存在
3. **点击位置不准**：可能需要调整游戏分辨率或识别参数
4. **OCR识别慢**：首次加载模型需要时间，后续识别会加快

### 调试信息
- 查看主界面日志框获取实时运行状态
- 悬浮窗显示当前识别到的文字和边界框
- 可通过修改日志级别获取更详细的调试信息
